Binary Heap

A Binary Heap could be a Binary Tree with following properties. It's an entire tree (All levels are completely filled except possibly the last level and therefore the last level has all keys as left as possible). This property of Binary Heap makes them suitable to be stored in an array. A Binary Heap is either Min Heap or Max Heap. in a very Min Binary Heap, the key at root must be minimum among all keys present in Binary Heap. the identical property must be recursively true for all nodes in Binary Tree. Max Binary Heap is comparable to MinHeap.

Operations on Min Heap:

  • getMini(): It returns the foundation element of Min Heap. Time Complexity of this operation is O(1).
  • extractMin(): Removes the minimum element from MinHeap. Time Complexity of this Operation is O(Logn) as this operation has to maintain the heap property (by calling heapify()) after removing root.
  • decreaseKey(): Decreases value of key. The time complexity of this operation is O(Logn). If the decreases key value of a node is bigger than the parent of the node, then we don't must do anything. Otherwise, we want to traverse up to mend the violated heap property.
  • insert(): Inserting a replacement key takes O(Logn) time. We add a replacement key at the tip of the tree. IF new key's greater than its parent, then we don't must do anything. Otherwise, we want to traverse up to mend the violated heap property.
  • delete(): Deleting a key also takes O(Logn) time. We replace the key to be deleted with minum infinite by calling decreaseKey(). After decreaseKey(), the minus infinite value must reach root, so we call extractMin() to get rid of the key.

    Applications of Heaps:

  • Heap Sort: Heap Sort uses Binary Heap to sort an array in O(nLogn) time.
  • Priority Queue: Priority queues will be efficiently implemented using Binary Heap because it supports insert(), delete() and extractmax(), decreaseKey() operations in O(logn) time. Binomoial Heap and Fibonacci Heap are variations of Binary Heap. These variations perform union also efficiently.
  • Graph Algorithms: The priority queues are especially employed in Graph Algorithms like Dijkstra's Shortest Path and Prim's Minimum Spanning Tree.
  • Many problems will be efficiently solved using Heaps. See following for instance.a) K'th Largest Element in an array.b) Sort an almost sorted array/c) Merge K Sorted

    • Applications of Heaps:

    Tries, also known as prefix trees, are tree-based data structures primarily used for efficient retrieval of strings. They are especially useful when dealing with dictionary-like applications, where words or keys need to be stored and searched for quickly. In a trie, each node represents a prefix or a complete string, and the edges represent characters. By traversing down the trie, one can search for a specific string or find all strings with a common prefix efficiently. Tries are commonly used in applications like autocomplete, spell-checking, and IP routing. In summary, heaps are tree-based data structures used to efficiently implement priority queues, while tries are tree-based data structures used for efficient string retrieval, particularly in applications that involve searching for strings or prefixes.
